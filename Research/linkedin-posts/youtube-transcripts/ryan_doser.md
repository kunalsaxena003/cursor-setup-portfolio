#Transcript
Introduction

In this video, the creator explains a Claude Code SEO workflow that takes a YouTube video URL and converts it into a fully SEO-optimized blog post in a single step. The process takes about 60 seconds and publishes directly to a website without manual copying or pasting. The output is structured, formatted, and optimized automatically, matching the quality of a manually written SEO article. The video also mentions that the SEO skill used in this workflow is available for free through the creator’s guide, along with other AI tools and automations.

Claude Code SEO workflow prerequisites

Before running this workflow, there are a few setup requirements. The creator recommends using Claude Code through an IDE like VS Code, although other options like Cursor or terminal access also work. For usage, a higher-tier plan such as Claude Max is preferred because lower plans may not provide enough tokens at scale. Another key requirement is having a custom SEO blog post writer skill. This skill is not generic and is trained on specific writing style, tone, and SEO rules, meaning users should adapt or recreate it rather than copy it directly.

MCP connection with WordPress

A critical part of the workflow is connecting Claude Code to a website through MCP. In this case, WordPress is used with a plugin that enables this integration. This allows Claude Code to directly create and upload blog posts as drafts or published content. The website used in the example has no backlinks or authority, making it a testing ground to evaluate pure SEO performance without external influence.

Blotato for scraping YouTube transcripts

To generate content, the system needs access to the YouTube video transcript. This is handled using a tool called Blotato, which can scrape transcripts via an MCP integration. Once connected, Claude Code can pull transcript data automatically. While Blotato is used in this example, any tool capable of extracting YouTube transcripts can work as long as it integrates into the workflow.

Claude Code settings tips

Some configuration choices improve efficiency. The recommended model is Sonnet 4.6, as more powerful models are unnecessary and consume more tokens. Extended thinking mode should be turned off to avoid excessive usage. The creator also keeps a local archive folder where every generated blog post is saved. This serves as a reference and helps refine the SEO writing skill over time by creating a feedback loop.

Running the SEO workflow

The workflow begins by providing a YouTube video URL, along with optional inputs like target keywords, secondary keywords, and a title. These inputs improve the quality of the generated output. Once executed, Claude Code processes the transcript, applies the SEO writing skill, and generates a complete blog post. The system then saves a local copy and uploads the draft directly to WordPress via MCP.

Reviewing the blog post output

The generated blog post follows standard SEO practices. The target keyword appears in the introduction and headings. The original YouTube video is automatically embedded within the article. Relevant external links are added, and internal links can be included as part of the skill configuration. The structure, formatting, and readability are handled automatically without manual editing.

AI screenshot feature

One of the more advanced features is the ability to extract screenshots from the YouTube video. The system identifies relevant timestamps, captures frames, compresses them, and inserts them into the blog post with optimized alt text. These images are placed contextually within the article, improving both user experience and SEO without additional effort.

Featured image auto extract

The workflow also pulls the YouTube thumbnail and assigns it as the featured image of the blog post. This removes the need for manual image selection and ensures consistency between the video and the article.

WordPress SEO settings

Claude Code also configures key SEO elements داخل WordPress automatically. It generates an optimized permalink, assigns categories based on content, sets a meta description, and fills in SEO plugin fields like focus keywords. Author selection and other publishing details are also handled within the workflow.

Indexing with Google Search Console

After publishing, the final step is submitting the URL to Google Search Console for indexing. While some search engines can be automated, manual submission is still required in this case. This ensures the page is crawled and indexed quickly.

Real world SEO rankings on Google

The creator demonstrates early results from this workflow. Despite the website having zero backlinks and no authority, newly created blog posts begin ranking on page one for certain keywords within days. Examples include rankings for low to moderate competition terms, as well as cases where both the YouTube video and the blog post appear together in search results.

SEO credibility and client results

To support credibility, the creator references experience managing large-scale SEO projects with millions of clicks and impressions. This highlights that the workflow is not presented as a shortcut or guaranteed solution, but rather as a scalable system that can complement existing SEO strategies.

Final thoughts

The main takeaway is that this workflow automates the entire process of turning video content into SEO-optimized blog posts. It combines transcript extraction, AI writing, media generation, and direct publishing into a single pipeline. While the system is highly efficient, the creator emphasizes the importance of human review, customization of the SEO skill, and continuous improvement to achieve consistent results.
